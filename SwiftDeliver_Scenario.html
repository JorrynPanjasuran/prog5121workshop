Scenario Overview — SwiftDeliver

SwiftDeliver is a console-based messaging application developed for a food delivery service company operating across multiple branches in South Africa. Kitchen managers, branch supervisors, and delivery riders use the application to exchange real-time coordination messages about orders, from the moment a meal is prepared in the kitchen through to delivery confirmation at the customer's address.

The application is built entirely as a Java console application with no graphical user interface. All user interaction happens through typed input and printed output. This mirrors the original POE specification requirement that no GUIs or JOptionPane dialogs may be used.

The scenario demonstrates the same coding constructs required by the POE across all three parts: variables and variable scope, data types, classes and methods, operators, decisions, loops, and arrays. All five test messages are drawn from realistic food delivery communications.

Part 3 — Store Data and Display Reports

100 Marks

Part 3 extends SwiftDeliver with five in-memory arrays, a fourth menu option for stored messages, and a full suite of unit tests. It also requires reading the JSON file back into an array on startup.

3.1 Required Arrays

Array NameContents
Sent MessagesContains all messages the user chose to send.
Disregarded MessagesContains all messages the user chose to disregard.
Stored MessagesLoaded from the messages.json file on startup, then updated as new messages are stored. Contains all stored messages.
Message HashesParallel array containing all message hashes in the same order as the sent messages array.
Message IDsParallel array containing all message IDs in the same order as the sent messages array.

3.2 Fourth Menu Option — Stored Messages Sub-Menu

OptionFeatureExpected Behaviour
aDisplay sender and recipient of all stored messagesLoops over the stored messages array and prints each recipient and message body.
bDisplay the longest stored messageFinds and prints the message with the greatest character count from the stored array.
cSearch by Message IDAccepts a message ID from the user, searches the sent and stored arrays, returns the matching recipient and message.
dSearch by recipientAccepts a recipient number, returns all messages in the sent and stored arrays for that recipient.
eDelete by Message HashAccepts a hash, removes the matching message from the stored array and compacts the array. Returns a confirmation message.
fDisplay full reportPrints the Message Hash, Recipient, and Message body for every stored message.

// I pre-load the 5 SwiftDeliver test messages specified in my Part 3 scenario.
public static void populateTestMessages() {

    // Message 1 Sent: kitchen manager asks if the pizza has left
    Message msg1 = new Message("+27761234567", "Has the pizza left the kitchen yet?", sentCount);
    msg1.setMessageType("sent");
    sentMessages[sentCount] = msg1;  messageHashes[sentCount] = msg1.getMessageHash();
    messageIDs[sentCount]   = msg1.getMessageID();   sentCount++;

    // Message 2 Stored: rider notifies customer about a traffic delay (longest message)
    Message msg2 = new Message("+27769876543",
        "Your order is stuck in traffic on the N1, estimated arrival is now 8:15 PM.", storeCount);
    msg2.setMessageType("stored");
    storedMessages[storeCount] = msg2;   storeCount++;

    // Message 3 Disregarded: wrong order notification, no longer relevant
    Message msg3 = new Message("+27761112233", "Oops, wrong order sent to table six.", discardCount);
    msg3.setMessageType("disregarded");
    disregardedMessages[discardCount] = msg3;   discardCount++;

    // Message 4 Sent: rider confirms arrival (local format number as per POE table)
    Message msg4 = new Message("0769876543", "Rider is outside now!", sentCount);
    msg4.setMessageType("sent");
    sentMessages[sentCount] = msg4;  messageHashes[sentCount] = msg4.getMessageHash();
    messageIDs[sentCount]   = msg4.getMessageID();   sentCount++;

    // Message 5 Stored: customer confirms delivery was successful
    Message msg5 = new Message("+27769876543", "Order confirmed and signed for, thanks!", storeCount);
    msg5.setMessageType("stored");
    storedMessages[storeCount] = msg5;   storeCount++;
}
